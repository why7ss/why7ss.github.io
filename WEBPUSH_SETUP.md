🔔 Инструкция по настройке Web Push Notifications

========================================
1. ГЕНЕРАЦИЯ VAPID КЛЮЧЕЙ
========================================

Для работы Web Push уведомлений нужны VAPID ключи. Сгенерируйте их:

cd backend
npm run generate-vapid

Или вручную:
node src/scripts/generateVapidKeys.js

Скопируйте вывод и добавьте в ваш .env файл.

========================================
2. ПЕРЕМЕННЫЕ ОКРУЖЕНИЯ (.env)
========================================

Добавьте эти переменные в backend/.env:

VAPID_PUBLIC_KEY=ваш_публичный_ключ
VAPID_PRIVATE_KEY=ваш_приватный_ключ
VAPID_SUBJECT=mailto:admin@easygame.com

Пример:
VAPID_PUBLIC_KEY=BEl62iUeJBIubNgTi5K...
VAPID_PRIVATE_KEY=aCEjsH2Yy5qX...
VAPID_SUBJECT=mailto:admin@easygame.com

========================================
3. ПЕРЕМЕННЫЕ FRONTEND (если нужны)
========================================

В frontend/.env (если требуется):
VITE_API_URL=http://localhost:5000

========================================
4. УСТАНОВКА npm ПАКЕТОВ
========================================

Backend:
cd backend
npm install web-push

Frontend: (уже должен быть настроен)

========================================
5. КАК РАБОТАЕТ
========================================

БРАУЗЕР (Frontend):
1. При первом посещении просит разрешение на уведомления
2. Пользователь нажимает "Включить уведомления"
3. Service Worker подписывается на push notifications
4. Service Worker отправляет subscription на сервер

СЕРВЕР (Backend):
1. Backend сохраняет subscription пользователя в БД
2. Когда админ отправляет уведомление через API
3. Backend отправляет Web Push всем подписчикам

ПОЛЬЗОВАТЕЛЬ:
1. Получает push notification на компьютер (даже если вкладка закрыта)
2. Кликает на уведомление - открывается приложение

========================================
6. ТЕСТИРОВАНИЕ
========================================

1. Запустите backend:
   npm run dev

2. Откройте приложение в браузере
3. Разрешите уведомления
4. Откройте админ-панель → Уведомления
5. Отправьте уведомление
6. Должны придти:
   - Socket.IO уведомление (если вкладка открыта)
   - Web Push уведомление на компьютер (даже если вкладка закрыта!)

========================================
7. DEBUGGING
========================================

Откройте DevTools (F12):
- Console (Консоль) - смотрите логи
- Application → Service Workers - проверьте регистрацию
- Network - смотрите запросы

Backend логи покажут:
✅ Push отправлена пользователю ...
❌ Ошибка при отправке...
